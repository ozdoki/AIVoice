# 実装ルール

- OS 依存の処理、低レベル音声処理、文字注入は Rust 側に寄せ、UI 層に直接持ち込まない。
- 音声認識プロバイダは差し替え前提の境界を保ち、ベンダー固有実装をアプリ全体へ漏らさない。
- Windows 先行の判断は許容するが、不要に将来の他 OS 対応を閉ざす設計は避ける。
- 新しい依存追加は必要性を明示できる場合に限定し、既存の責務分離を崩さない。
- UI は常駐トレイ、設定、軽量な状態表示を優先し、MVP 範囲外の管理画面化を避ける。
